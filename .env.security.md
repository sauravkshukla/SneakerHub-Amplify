# Environment Variables Security Guide

## ⚠️ CRITICAL SECURITY NOTICE

**NEVER commit `.env` files to Git!**

This repository uses environment variables to store sensitive configuration. These files are excluded from Git via `.gitignore`.

## Files That Should NEVER Be Committed

- `frontend/.env` - Contains API keys and secrets
- `backend/src/main/resources/application.properties` - Contains database credentials and JWT secrets
- Any file with actual API keys, passwords, or tokens

## Safe Files (Can Be Committed)

- `frontend/.env.example` - Template without real values
- `backend/src/main/resources/application.properties.example` - Template without real values

## Setup Instructions

### Frontend Setup

1. Copy the example file:
   ```bash
   cp frontend/.env.example frontend/.env
   ```

2. Edit `frontend/.env` and add your real values:
   ```
   VITE_API_URL=http://localhost:8000
   VITE_GEMINI_API_KEY=your_actual_api_key_here
   ```

### Backend Setup

1. Copy the example file:
   ```bash
   cp backend/src/main/resources/application.properties.example backend/src/main/resources/application.properties
   ```

2. Edit `application.properties` and configure:
   - Database credentials
   - JWT secret (use a strong random string)
   - Email configuration (if needed)

## Getting API Keys

### Google Gemini API Key
1. Visit: https://makersuite.google.com/app/apikey
2. Create a new API key
3. Add to `frontend/.env` as `VITE_GEMINI_API_KEY`

## Production Deployment

### Environment Variables for Production

**Frontend (Vercel/Netlify):**
- Set environment variables in the hosting platform dashboard
- Never hardcode secrets in code

**Backend (Railway/Heroku/AWS):**
- Use platform-specific environment variable management
- Generate strong JWT secret: `openssl rand -base64 64`
- Use managed database services with secure credentials

## Security Checklist

- [ ] `.env` files are in `.gitignore`
- [ ] No API keys in code
- [ ] No passwords in code
- [ ] JWT secret is strong and random
- [ ] Database credentials are secure
- [ ] Production uses different secrets than development
- [ ] API keys have appropriate restrictions/scopes

## If You Accidentally Committed Secrets

1. **Immediately revoke/regenerate** all exposed credentials
2. Remove from Git history:
   ```bash
   git filter-branch --force --index-filter \
   "git rm --cached --ignore-unmatch frontend/.env" \
   --prune-empty --tag-name-filter cat -- --all
   ```
3. Force push (if safe to do so)
4. Update all team members
